How it works, briefly

A claim flows through one governed pipeline. It arrives at an n8n webhook, an OpenAI model returns a strict, schema bound assessment (type, severity, a fraud score, an FCA vulnerability flag and a routing recommendation), and the claim is written to PostgreSQL as the system of record. A deterministic rule, not the model, decides accept, refer or decline, and routes to the right team. The guiding principle throughout is simple: the model assists, but a person and an explicit rule decide every outcome.

One point worth making plainly: I already had n8n self hosted, so instead of using a trial environment I built the workflow directly on my own infrastructure, the NAVADA Edge network, with PostgreSQL and Grafana behind it. On top of that I built the web application itself, Ava and every page you will see. It is one connected system on real, owned infrastructure.
